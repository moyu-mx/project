## 功能概览

| 模块 | 说明 |
| --- | --- |
| 数据清洗 | 读取 `data.csv`，删除无效列、统一日期格式、派生时间维度字段 |
| 数据分析 | 利润、销售额、客单价、区域、淡旺季、新老客户、RFM 等 **17 张**图表 |
| 数据入库 | 星型模型写入 SQLite，含年度/区域/RFM 等预计算汇总表 |
| Web 仪表盘 | 固定导航 + ECharts 动态图表，点击放大并展示数据解读 |
| Agent 报告 | Agent1 月/周/日报、商品/用户 Markdown；Agent2 波动原因、异常预警、机会点 |
| 大模型查询 | MCP 工具注入 schema 语义，支持 **local / API** 双模式 NL2SQL |

### 功能与运行模式

| 功能 | local 模式（默认） | api 模式 |
| --- | --- | --- |
| Web 仪表盘、图表 | 可用 | — |
| Agent1 分析报告 | 可用 | — |
| Agent2 数据洞察 | 规则模板生成 | 大模型生成文案 |
| 智能查询 NL2SQL | 本地规则匹配 | MCP + 大模型推理 |

默认 **local 模式**可完全离线运行。API 模式配置见 [环境配置与运行 · 关键配置](./环境配置与运行.md#关键配置)。

## 项目结构

```
prj/
├── data.csv                    # 原始数据（约 5 万行）
├── data/cleaned/               # 清洗后 CSV
├── data/superstore.db          # SQLite（ETL 生成，Web 必需）
├── src/
│   ├── cleaning/               # 数据清洗
│   ├── analysis/               # 分析脚本与图表
│   ├── etl/                    # 入库 ETL
│   ├── db/                     # 数据库连接与查询执行
│   ├── agents/                 # Agent1/2 报告与洞察
│   ├── mcp/                    # MCP 工具与 schema 语义
│   └── llm/                    # NL2SQL、SQL 校验
├── web/                        # FastAPI 入口、模板、静态资源
├── sql/schema.sql              # 建表 DDL
├── config/                     # db.yaml、llm.yaml、MCP 配置
├── prompts/nl2sql.txt          # LLM 提示词模板
├── packaging/superstore.spec   # PyInstaller 打包配置
├── scripts/build_exe.ps1       # 一键打包脚本
├── run_app.py                  # exe 启动入口
├── results/charts/             # 分析图表输出
└── tests/                      # 单元测试与冒烟测试
```

### 数据库表结构

默认 SQLite 路径：`data/superstore.db`，表结构见 `sql/schema.sql`：

- 维度表：`customers`、`products`、`regions`
- 事实表：`orders`
- 汇总表：`agg_sales_by_year`、`agg_sales_by_region_year`、`customer_rfm` 等

## Web 使用说明

访问地址：**http://127.0.0.1:8000**（需先按 [环境配置与运行](./环境配置与运行.md) 完成部署并启动服务）

- **左侧导航**：固定不随页面滚动，点击跳转到对应分析区块
- **图表交互**：点击任意图表放大查看，右侧展示基于数据库的动态分析解读
- **智能查询**：页面底部输入自然语言问题，系统自动生成 SQL 并展示查询结果
- **Agent1 分析报告**：选择报告类型与周期，生成含 Mermaid 图表的 Markdown（建议 Typora 或 GitHub 预览）
- **Agent2 数据洞察**：指定年月，自动生成波动原因、异常预警、机会点报告

### 智能查询示例

- `2013年各区域销售额是多少？`
- `各年销售额`
- `客单价趋势`
- `RFM客户价值分布`

## 大模型对话（MCP 双模式）

### 架构

```
用户提问 → chat_engine
              ├─ local：MCP 工具加载 schema + 本地规则生成 SQL
              └─ api：OpenAI 兼容 API + MCP function calling 多轮推理
           → SQL 安全校验 → 执行 → 返回结果
```

### MCP 工具

| 工具 | 作用 |
| --- | --- |
| `list_tables` | 列出所有表及中文说明 |
| `get_database_schema` | 表结构、字段类型、中文含义、业务规则 |
| `get_field_glossary` | market/segment/category 等易混淆字段对照 |
| `preview_table` | 预览表样本数据 |
| `validate_sql_query` | 校验 SQL 是否仅含安全的 SELECT |

字段语义定义见 `src/mcp/schema_catalog.py`。

### 模式说明

- **local（默认）**：MCP 加载 schema + 本地规则匹配，无需 API Key
- **api**：OpenAI 兼容接口 + MCP function calling，需配置密钥

模式切换与密钥配置见 [环境配置与运行 · 关键配置](./环境配置与运行.md#关键配置)。

### 独立 MCP Server

供 Cursor 等客户端连接，配置与启动方式见 [环境配置与运行 · Cursor MCP 配置](./环境配置与运行.md#cursor-mcp-配置可选)。

## 数据清洗说明

| 步骤 | 处理内容 |
| --- | --- |
| 删除列 | 移除空值过多的 `Postal Code` |
| 日期 | 兼容 `M/D/YYYY` 与 `DD-MM-YYYY` 两种格式 |
| 数值 | 转换 Sales 等字段，剔除无效行（约 23 条） |
| 派生字段 | `Order-year`、`Order-month`、`quarter`、`Ship-year`、`Ship-month` |

清洗报告：`results/cleaning_report.json` · ETL 校验：`results/etl_validation.json`
