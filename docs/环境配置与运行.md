## 环境与部署需求

### 硬件与系统

| 项目 | 最低要求 | 说明 |
| --- | --- | --- |
| 操作系统 | Windows 10/11 64 位 | 开发与打包脚本为 PowerShell；macOS/Linux 可跑 Python 源码，打包脚本需自行适配 |
| 内存 | 4 GB | 建议 8 GB（pandas + sklearn 分析阶段占用较高） |
| 磁盘 | 2 GB 可用空间 | 含虚拟环境、构建缓存、`data.csv`（约 12 MB）及数据库 |
| 网络 | 非必须 | **local 模式**完全离线；**API 模式**需访问 DeepSeek 等兼容接口 |

### 软件依赖

| 软件 | 版本 | 用途 |
| --- | --- | --- |
| Python | **3.10 ~ 3.12** | 开发与打包 |
| Git | 任意近期版本 | 克隆仓库（可选） |
| 浏览器 | Chrome / Edge 最新版 | 访问 Web 仪表盘 |

Python 包见 `requirements.txt`；**打包**额外需要 `requirements-build.txt`（PyInstaller）。

### 两种运行方式对比

| | 开发模式（源码） | 打包模式（exe） |
| --- | --- | --- |
| 适用场景 | 开发、调试、改代码 | 无 Python 环境的机器上直接运行 |
| 前置条件 | 安装 Python + venv + 跑完流水线 | 在本机完成打包 |
| 启动命令 | `uvicorn web.app:app ...` | 双击 `SuperstoreAnalytics.exe` |
| 改代码后 | 保存即生效（`--reload`） | 需重新打包 |
| 数据库位置 | `data/superstore.db` | exe 同级 `data/superstore.db` |

### 首次部署检查清单

在启动 Web 或打包前，确认以下文件/目录已生成：

```
[ ] .venv/                            # 虚拟环境
[ ] data/cleaned/superstore_clean.csv # 清洗结果
[ ] data/superstore.db                # ETL 入库（Web 必需）
[ ] results/charts/*.png              # 17 张分析图（打包时会复制）
[ ] results/forecast_report.json      # 预测报告
[ ] results/cleaning_report.json      # 清洗报告
[ ] pytest tests/ -q 全部通过         # 建议
```

## 关键配置

项目配置文件位于 `config/`，其中与运行直接相关的如下。

### 配置一览

| 文件 | 是否必须修改 | 作用 |
| --- | --- | --- |
| `config/db.yaml` | 否 | SQLite 数据库路径，默认 `data/superstore.db` |
| `config/llm.yaml` | 否 | 大模型模式（`local` / `api`）、模型名称、接口地址 |
| `config/llm_keys.yaml` | **仅 API 模式需要** | 存放 API 密钥（**不提交 Git**，见 `.gitignore`） |
| `config/mcp.cursor.json` | 否 | Cursor IDE 连接 MCP Server 的示例配置 |

### API 密钥获取与配置

仅在 Web 页面切换到 **api** 模式时需要配置密钥。项目使用 **OpenAI 兼容接口**，默认对接 DeepSeek，也可替换为其他兼容服务（如 SiliconFlow、OpenAI 等）。

#### 方式一：密钥文件（推荐）

```powershell
cd e:\26实训\prj
copy config\llm_keys.yaml.example config\llm_keys.yaml
```

编辑 `config/llm_keys.yaml`：

```yaml
api_key: sk-你的密钥
```

#### 方式二：环境变量

任选其一设置（优先级低于 `llm_keys.yaml`）：

```powershell
# DeepSeek（与 config/llm.yaml 默认 provider 对应）
$env:DEEPSEEK_API_KEY = "sk-你的密钥"

# 通用别名
$env:LLM_API_KEY = "sk-你的密钥"

# SiliconFlow 等第三方兼容接口
$env:SILICONFLOW_API_KEY = "sk-你的密钥"
```

#### 密钥从哪里获取？

| 服务 | 注册地址 | 说明 |
| --- | --- | --- |
| **DeepSeek**（默认） | https://platform.deepseek.com | 注册后在「API Keys」页面创建密钥；`config/llm.yaml` 中已配置 `base_url: https://api.deepseek.com`、`model: deepseek-chat` |
| SiliconFlow | https://cloud.siliconflow.cn | 需同步修改 `config/llm.yaml` 中的 `base_url` 与 `model` |
| OpenAI | https://platform.openai.com | 需将 `provider`/`model`/`base_url` 改为 OpenAI 对应值 |

#### 密钥加载优先级

```
config/llm_keys.yaml  →  环境变量（DEEPSEEK_API_KEY / LLM_API_KEY / SILICONFLOW_API_KEY）  →  config/llm.yaml 内 api.api_key
```

#### 切换到 API 模式

编辑 `config/llm.yaml`：

```yaml
mode: api
api_enabled: true

api:
  provider: deepseek
  model: deepseek-chat
  base_url: https://api.deepseek.com
```

重启 Web 服务后，页面「调用方式」选择 **api** 即可。

#### 验证密钥是否生效

```powershell
.\.venv\Scripts\python scripts\test_api_mode.py
# 输出 has_key: True 表示密钥已加载

.\.venv\Scripts\python scripts\test_siliconflow_ping.py
# 测试 API 连通性（需网络）
```

### 数据库路径

`config/db.yaml` 默认指向项目内 SQLite 文件，一般无需修改：

```yaml
database:
  url: "sqlite:///data/superstore.db"
```

路径相对于项目根目录；打包后 exe 同级目录下同样生效。

### Cursor MCP 配置（可选）

若使用 Cursor 连接本项目的 MCP Server，复制 `config/mcp.cursor.json` 到 Cursor 设置，并将 `cwd` 改为本机项目路径：

```json
{
  "mcpServers": {
    "superstore-analytics": {
      "command": "python",
      "args": ["-m", "src.mcp.server"],
      "cwd": "e:/26实训/prj"
    }
  }
}
```

独立启动 MCP Server：

```powershell
.\.venv\Scripts\python -m src.mcp.server
```

## 快速开始（首次部署）

> 将 `e:\26实训\prj` 替换为你本机克隆后的项目路径。

### 1. 环境准备

```powershell
cd e:\26实训\prj
python -m venv .venv
.\.venv\Scripts\Activate.ps1
.\.venv\Scripts\pip install -r requirements.txt
```

若 PowerShell 禁止执行脚本，先运行：

```powershell
Set-ExecutionPolicy -Scope CurrentUser RemoteSigned
```

### 2. 数据处理流水线

**首次部署必须按顺序执行**（生成数据库与图表；Web 依赖 `data/superstore.db`）：

```powershell
# 数据清洗
.\.venv\Scripts\python -m src.cleaning.clean_data

# 生成全部分析图表 + 预测（内含清洗，可单独跑此命令替代上一行）
.\.venv\Scripts\python -m src.analysis.run_all

# 写入 SQLite（Web / NL2SQL / Agent 的数据源）
.\.venv\Scripts\python -m src.etl.load_to_db
```

流水线耗时参考：清洗 + 分析约 1~3 分钟（视机器性能而定）。

### 3. 启动 Web 服务

```powershell
.\.venv\Scripts\python -m uvicorn web.app:app --reload --app-dir . --port 8000
```

浏览器访问：**http://127.0.0.1:8000**

### 4. 验证安装

```powershell
# 自动化测试（17 张图、NL2SQL、Agent 等）
.\.venv\Scripts\python -m pytest tests/ -q

# 手动检查：首页能打开、至少一张图表有数据、智能查询能返回结果
```

## 日常启动

**已完成首次部署**后，日常只需：

```powershell
cd e:\26实训\prj
.\.venv\Scripts\Activate.ps1
.\.venv\Scripts\python -m uvicorn web.app:app --reload --app-dir . --port 8000
```

### 何时需要重新跑流水线？

| 场景 | 需要执行的命令 |
| --- | --- |
| 仅改 Web 前端 / API 逻辑 | 重启 uvicorn 即可 |
| 修改 `src/analysis/` 出图逻辑 | `run_all`（必要时再 `load_to_db`） |
| 修改 `src/cleaning/` 或 `data.csv` | `clean_data` → `run_all` → `load_to_db` |
| 修改 `sql/schema.sql` 或 ETL | `load_to_db`（若清洗逻辑变了则从头跑） |
| 修改 `src/mcp/schema_catalog.py` | 重启 uvicorn（NL2SQL 语义即时生效） |

> **注意**：执行 `load_to_db` 前必须先 **Ctrl+C 停止 Web**，否则 Windows 下 SQLite 文件被占用会报错。

## 打包与分发

可将项目打包为 Windows 可执行目录，便于在没有 Python 环境的机器上运行。

### 打包前置条件

1. 开发模式首次部署已完成（存在 `data/superstore.db`）
2. 已执行 `run_all` 生成 `results/charts/` 与 `forecast_report.json`
3. 虚拟环境中已安装运行依赖

### 打包命令

```powershell
cd e:\26实训\prj
.\.venv\Scripts\Activate.ps1
.\scripts\build_exe.ps1
```

脚本会自动：

1. 安装 `requirements-build.txt`（PyInstaller）
2. 清理旧的 `build/`、`dist/`
3. 执行 `packaging/superstore.spec` 构建
4. 将数据库、图表、配置复制到 `dist/SuperstoreAnalytics/`
5. 生成 `dist/SuperstoreAnalytics/README.txt`

### 分发说明

将整个 **`dist/SuperstoreAnalytics/` 文件夹**压缩分发（不要只发 exe 单文件）。

使用步骤：

1. 解压到任意目录（路径尽量不含中文与空格）
2. 双击 **`SuperstoreAnalytics.exe`**
3. 浏览器自动打开 http://127.0.0.1:8000
4. 关闭控制台窗口即停止服务

可选参数：

```text
SuperstoreAnalytics.exe --port 8001
SuperstoreAnalytics.exe --no-browser
SuperstoreAnalytics.exe --host 0.0.0.0 --port 8000   # 局域网访问（需谨慎）
```

### API Key（打包版）

默认 **local 模式**，无需密钥。若需 API 模式，在 exe 同级 `config/` 下按 [关键配置](#关键配置) 一节操作。

## 代码更新后重新打包

修改代码并验证通过后，按以下流程重新发布 exe：

```powershell
cd e:\26实训\prj
.\.venv\Scripts\Activate.ps1

# 1. 停止正在运行的 Web / exe（Ctrl+C 或关闭控制台）

# 2. 若改动涉及数据或分析，重新跑流水线
.\.venv\Scripts\python -m src.analysis.run_all      # 分析有变时
.\.venv\Scripts\python -m src.etl.load_to_db        # 库表或 ETL 有变时

# 3. 回归测试
.\.venv\Scripts\python -m pytest tests/ -q

# 4. 重新打包
.\scripts\build_exe.ps1

# 5. 冒烟验证：在 dist 目录独立运行 exe，检查首页、图表、查询、Agent
cd dist\SuperstoreAnalytics
.\SuperstoreAnalytics.exe
```

### 改动类型与打包注意点

| 改动位置 | 测试重点 | 打包注意 |
| --- | --- | --- |
| `web/templates/` · `web/static/` | 页面布局、交互 | 模板/静态资源已打入 spec，重新 build 即可 |
| `web/app.py` · `src/llm/` | API、NL2SQL | 同上 |
| `src/analysis/` | 图表 PNG、预测 | 打包前重新 `run_all`，脚本会复制最新 `results/charts/` |
| `src/etl/` · `sql/schema.sql` | 数据库结构 | 打包前重新 `load_to_db`，脚本会复制最新 `superstore.db` |
| `config/llm.yaml` | 默认模式 | 打包后 `config/` 会复制到 dist，可按需修改 |
| `requirements.txt` 新增依赖 | import 是否遗漏 | 若 PyInstaller 漏打包，在 `packaging/superstore.spec` 的 `hiddenimports` 补充 |

## 常见问题

**Q：ETL 报错「另一个程序正在使用此文件」**  
A：先 `Ctrl+C` 停止 Web 服务或关闭 exe，再执行 `load_to_db`。

**Q：端口 8000 被占用**

```powershell
netstat -ano | findstr :8000
taskkill /PID <PID> /F
# 或换端口
.\.venv\Scripts\python -m uvicorn web.app:app --reload --app-dir . --port 8001
```

**Q：页面未更新**  
A：重启服务后使用 `Ctrl + F5` 强制刷新；静态资源已带版本号，若仍缓存可清浏览器缓存。

**Q：API 模式提示缺少密钥**  
A：按 [关键配置](#关键配置) 一节创建 `config/llm_keys.yaml` 或设置环境变量。若暂不需要大模型能力，保持 `mode: local` 即可。

**Q：打包后 exe 启动报错找不到数据库**  
A：确认打包前已执行 `load_to_db`；检查 `dist/SuperstoreAnalytics/data/superstore.db` 是否存在；不要只复制 exe 而不复制整个文件夹。

**Q：修改代码后 exe 行为未变**  
A：exe 不会自动同步源码改动，必须重新执行 [代码更新后重新打包](#代码更新后重新打包) 流程。

**Q：路径含中文是否影响运行？**  
A：开发模式一般可用；打包分发建议目标路径使用英文，减少个别环境下的编码问题。
